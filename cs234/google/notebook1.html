1. Accessing the History page¶

The Chrome browser keeps track of interactions you perform on the Chrome browser:

  • searches you perform on the "location" bar
  • links you open either directly or by clicking on links on a page

All this information is stored in a file named 'History'. In the Mac computer, you can access
this file by going to this local URL (after replacing USERNAME with your own):

file:///Users/USERNAME/Library/Application%20Support/Google/Chrome/Default/

On Windows, you should be able to do the same with the path:

C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default

If you download this file from this page in the browser, you can move it to the folder
that contains this notebook as well. Alternatively, once you make sure that it exists
and contains enough content, you can use the path to it instead.

If you open the file, at the very top you'll notice: SQLite format 3. That tells us that
this file is a database file that we can access in Python via the sqlite3 module.

2. The sqlite3 module¶

The most common way to store data is as tables in databases. There are many database systems out there, with the most used ones being relational database management systems, which make use of SQL (structured query language). However, NoSQL databases systems like MongoDB or Redis are also very popular.

SQLite is a database management system that doesn't require a server, which makes it different from more traditional systems like MySQL or Oracle.

To learn more about SQLite, I recommend their tutorial. Since we will not be creating our own databases in this class, we don't need to learn much about it. However, taking the CS 304 Databases course at some point is a good investment.

We will use Python's interface sqlite3 to talk to an SQLite file in order to read the content of it and submit different queries.

There is a very short (one page) tutorial in their website that shows how to talk to SQLite only to query data.

import datetime

def convertChromeTime(ms):
    """Convert the amount of microsends into a datetime object. 
    Google chrome doesn't use Unix epoch.
    """
    return datetime.datetime(1601, 1, 1) + datetime.timedelta(microseconds=ms)

Counting domain names¶

The algorithm:

  • get every URL and its count as it is shown in the urls table
  • extract the domain name (that leaves out specific parts of the URL)
  • update count of domain names
In [120]:
from collections import Counter

domainCounter = Counter()
cursor = conn.execute('select url, visit_count from urls')

for row in cursor.fetchall():
    url, count = row # grab the exisiting count for that URL
    domain = urlparse.urlparse(url).netloc
    domainCounter[domain] += count # accumulate URL counts
    
domainCounter.most_common(10)

Joining two tables¶

As we see in the tables above, the URL name is only in the urls table, while the two other tables contain an ID. This means that we have to look up the URL using the ID. A better way is to combine the two tables together, that is, to join them.

Joining is a topic in database theory, you can read more in this website: SQLite inner join.

For our two tables visits and urls, we'll do the following

SELECT urls.url, visit_time, visit_duration 
FROM visits
INNER JOIN urls on urls.id = visits.url

let's try it out:
