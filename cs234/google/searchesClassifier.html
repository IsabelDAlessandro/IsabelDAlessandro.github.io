Exploring features of search queries¶

In order to determine which queries are informational and which are navigational, the classifier will have to be trained using particualr features of these queries. Some simple features include the number of characters in the query, number of words in the query, and whether the query starts with a question word(like 'who',what','when',etc). Below is a simple function to extract these features from a given search phrase:

In [3]:
questionWords=['who','what','when','where','why','is','does','how','who','are']


def extractFeatures(phrase):
    my_dict={'num_chars': len(phrase),
             'num_words':len(phrase.split()),
             'question':phrase.split()[0].lower() in questionWords}
    return my_dict

Another feature of the search queries that we explored was the number of words of a particular part of speech(noun,verb, adjective, etc). We used the nltk.pos_tag function to extract the number of each part of speech in a given search phrase

def getPOS(phrase):
    tokens = nltk.word_tokenize(phrase.lower())
    text = nltk.Text(tokens)
    tagged = nltk.pos_tag(text)
    counts = Counter(tag for word,tag in tagged)
    for pos in postags:
        if pos not in counts.keys():
            counts[pos]=0
    total = sum(counts.values())
    return dict((word, float(count)/total) for word,count in counts.items())

This returns a dictionary with 45 parts of speech. We can try training the classifier using the counts of all of these parts of speech as features. However, because there are so many of them, we also run the risk of overwleming the other features, and of having a feature set that it too sparse. To get around this issue, let's also define another function to extract only the counts of three major parts of speech categories: noun, adjective, and verb:

In [51]:
def getPOS3(phrase):
    adj=['JJ','JJR','JJS']
    noun=['NN','NNP','NNS']
    verb=['VB','VBD','VBG','VBN','VBP','VBZ']
    tokens = nltk.word_tokenize(phrase.lower())
    text = nltk.Text(tokens)
    tagged = nltk.pos_tag(text)
    counts = Counter(tag for word,tag in tagged)
    for pos in postags:
        if pos not in counts.keys():
            counts[pos]=0
    anv_counts={'adj':counts['JJ']+counts['JJR']+counts['JJS'],
           'noun':counts['NN']+counts['NNP']+counts['NNS'],
           'verb':counts['VB']+counts['VBD']+counts['VBG']+counts['VBN']+counts['VBP']+counts['VBZ']}
    anv_counts_total=float(anv_counts['adj']+anv_counts['noun']+anv_counts['verb'])
    anv_norm={'adj':anv_counts['adj']/anv_counts_total, 'noun':anv_counts['noun']/anv_counts_total,
              'verb':anv_counts['verb']/anv_counts_total}
    return anv_norm

def getWebpage(phrase):
    """Returns a dictionary with the link, title, and snippet of each returned result"""
    #browser = webdriver.Chrome(executable_path='/Users/isabeldalessandro/Downloads/chromedriver')
    browser.get('http://google.com')
    inputBox = browser.find_element_by_name("q")
    inputBox.clear()
    inputBox.send_keys(str(phrase))
    inputBox.send_keys(Keys.ENTER)
    time.sleep(3)
    rawHtml = browser.page_source
    linkDictsList = map(extractLinkAndText, browser.find_elements_by_class_name("rc"))
    snippetsList = map(extractTextSnippet, browser.find_elements_by_class_name("st"))
    allInfoDictsList = []
    for i, dct in enumerate(linkDictsList):
        dct['snippet'] = snippetsList[i] # add the snippet to the link and title
        allInfoDictsList.append(dct)
    return allInfoDictsList

One idea that we had was to look at the simialrity of the query to the returned link/title/snippet. For example, a navigational search for 'facebook' would likely yield a top result with the title and url of the same name, while an informational search like 'what is the capital of north dakota' wouldn't have a top result with such a url or title. We decided to look only at the first result, since we assumed that a navigational search would be likely to return the intended website as the first result. To quantify the similarity between a query and it's results, we represented all of the strings(search query, title, url, snippet) as Tfidf vectors, and calculated the cosine similarity between the search query and each of the other three features.

In [14]:
stemmer = nltk.stem.porter.PorterStemmer()
remove_punctuation_map = dict((ord(char), None) for char in string.punctuation)

def stem_tokens(tokens):
    return [stemmer.stem(item) for item in tokens]

'''remove punctuation, lowercase, stem'''
def normalize(text):
    return stem_tokens(nltk.word_tokenize(text.lower().translate(remove_punctuation_map)))

vectorizer = TfidfVectorizer(tokenizer=normalize, stop_words='english')

def cosine_sim(text1, text2):
    """Calculate the cosine similairty between two texts """
    tfidf = vectorizer.fit_transform([text1, text2])
    return ((tfidf * tfidf.T).A)[0,1]

def extractFeatures4(phrase):
    my_dict={'num_chars': len(phrase),
             'num_words':len(phrase.split()),
             'question':phrase.split()[0].lower() in questionWords,
            'fullurlsim':cosine_sim(phrase,getWebpage(phrase)[1]['link']),#similarity of query and full url 
            'mainurlsim':cosine_sim(phrase,getWebpage(phrase)[1]['link'].split('/')[2]),#similariy of the query and first part of the url 
            'titlesim':cosine_sim(phrase,getWebpage(phrase)[1]['title']),#similariy of query and title 
            'snippetsim':cosine_sim(phrase,getWebpage(phrase)[1]['snippet'])}#similarity of query and snippet
    pos=getPOS3(phrase)
    my_dict.update(pos)
    return my_dict

To make sure that the classifier is distinguishing between informational and navigational queries based on their featrues (and not just based on their relative frequencies in the dataset), it is important to create balanced datasets, that is, datasets with equal numbers of navigational and informational queries. We will also divide the data up into three sets: a training set(~80% of the data), development set(~10% of the data), and testing set(~10% of the data)

%matplotlib inline

class_names=['navigational','informational']

def plot_confusion_matrix(cm, classes,
                          normalize=False,
                          title='Confusion matrix',
                          cmap=plt.cm.Blues):
    """
    This function prints and plots the confusion matrix.
    Normalization can be applied by setting `normalize=True`.
    """
    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]
        print("Normalized confusion matrix")
    else:
        print('Confusion matrix, without normalization')

    print(cm)

    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    fmt = '.2f' if normalize else 'd'
    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, format(cm[i, j], fmt),
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')
